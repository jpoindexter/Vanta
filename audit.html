Vision — full-capability, executive-function-first

Vanta is a FULL-CAPABILITY personal operator — it builds, ships, researches, runs companies, handles comms and calendar: everything Hermes does. The difference is the DESIGN LENS. Vanta is built executive-function-first: it supplies the initiation, sequencing, follow-through, working memory, and calm that every other agent assumes the human already brings. This is inclusive design, NOT accessibility-narrowing — the curb-cut effect: build for the people for whom finishing is hardest (executive-function disorders, ADHD, autism, dyslexia, dyscalculia, aphantasia — and anyone under stress, overload, or fatigue) and you build a better operator for EVERYONE. Capability is table stakes; the moat is follow-through — the first operator that helps you FINISH, not just do more. Grounded in Vanta's neurodivergent-first brain identity and the operator's own documented pattern (ideas-rich, fast to build, hard to finish, prone to bouncing and over-scoping). KEY FINDING: Hermes treats accessibility/ND as edge cases and Goose ignores it; BOTH assume the user supplies executive function. Vanta's wedge is to supply it.

The thesis

  • Capability is table stakes. Vanta matches Hermes's breadth (build / ship / run companies / research / comms). The differentiator is what happens to the HUMAN's executive function — not the model's capability.
  • Every agent today assumes the human supplies initiation, sequencing, and follow-through. Nobody has perfect executive function; ND people just hit that wall first and hardest — which is what makes the design failure visible.
  • Inclusive, not merely accessible — the curb-cut effect. Solve for the margin (executive-function disorders) and the result is better for everyone: under stress, overload, or fatigue, every brain becomes the edge case. Accessibility narrows; inclusivity is the real lens, and it aims at everyone.
  • The moat is follow-through: hold the thread, park the tangent, name the next step, get to DONE. Proof-of-concept already exists — Vanta's own anti-drift CLAUDE.md (PARKED.md, 'define Done', ship-ugly-first, one-feature-end-to-end, 'while-we're-at-it banned') is an executive-function prosthesis bolted onto a general agent. Make it native and universal.
  • Not a narrowing of scope — a sharpening of WHY. Vanta does everything a top operator does; it is simply the only one designed so that a brain that floods with ideas and stalls before the finish line still ships.

Fit filters (the design lens)

F1 Initiation friction — Reduces task activation cost — lowers the 'getting started' barrier (the core ADHD/exec-function wall).
F2 Externalized memory — Offloads working memory — remembers goals, context, where you left off, what was decided.
F3 Calm output — Low-arousal, low-firehose, sensory-regulating output — never a wall of noise.
F4 Low decision load — Sensible defaults; doesn't demand constant micro-choices (decision fatigue / analysis paralysis).
F5 Structure & follow-through — Breaks down, sequences, nudges, tracks — supports planning and follow-through deficits.
F6 Reliability / trust — Predictable, safe, honest — removes the vigilance burden. Safe-by-construction = ND feature.

Curation bias

RUTHLESS by design — but DROP means 'adds cognitive load / decision surface', NOT 'reduces capability'. Vanta keeps full operator breadth (build/ship/run/research/comms); what it drops is the noise (recipe marketplaces, MoA ensembles, integration sprawl, IDE adapters) that demands choices and attention without helping you finish. The audit's job is a SMALL curated pull-set + a LOT of explicit DROP. Default verdict is DROP unless a feature either (a) supplies executive function (≥2 EF filters, low cognitive cost) or (b) is core operator capability Vanta lacks. Note the live irony: six escalating 'add to goal' messages this session IS the bounce/over-scope pattern — exactly the thing Vanta should help contain. Result: ~8 worthy pulls, ~16 explicit drops. Goose: ~80% of distinct features DROP (dev-throughput). Hermes: capability kept, breadth-noise dropped, the EF signals it treats as edge cases become Vanta's spine.

CRUCIALLY, Hermes DOES have an accessibility/ND thread — but as edge cases, not the spine: voice-first interface ('can't type well, voice is huge' — motor accessibility), ADHD morning/evening standup workflows, proactive unprompted reminders (self-initiation support), memory persistence framed as cognitive-load reduction, and an NVDA screen-reader addon built by a blind user.
