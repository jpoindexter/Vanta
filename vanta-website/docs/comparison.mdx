# Vanta vs Hermes vs OpenClaw

Vanta, [Hermes Agent](https://github.com/NousResearch/hermes-agent) (Nous Research), and
[OpenClaw](https://github.com/openclaw/openclaw) are the same category: local, self-hosted
personal agents you run on your own hardware, model-provider-agnostic, reachable from your
own machine or a cheap VPS. They share most of their shape — CLI + TUI, messaging
gateways, skills, cross-session memory, subagents, scheduled jobs, browser and shell
control, MCP, an Electron desktop app.

:::note Honest by design
Because the three are so close, most rows below are **parity** (`≈`) — and we mark them as
parity, not as Vanta advantages. There are exactly three rows where Vanta is genuinely
different, and a few where Hermes or OpenClaw lead today. A comparison table is only worth
anything if you can trust it, so this one is built to survive a skeptic, not to win.
:::

## Where Vanta is genuinely different

Two are Vanta-only; the third is a local-first take on something Hermes also does:

1. **The boundary is a separate enforced layer, not a feature of the agent.** Vanta's
   [Rust kernel](/kernel) risk-classifies every action into allow / ask / block, enforces
   [scope containment](/safety-model), and writes a tamper-evident audit chain — and the
   agent loop _cannot bypass it_. Hermes and OpenClaw both have real controls (command
   approval, container isolation, allowlists), but those live inside the same process as
   the agent. Vanta's `assess()` is a gate the agent talks _through_, not _around_; the
   sandbox is the containment. _(Vanta-only.)_
2. **Goal-aware gating.** Vanta carries a goal ledger and dependency graph, so it knows
   the goal _before_ it chooses a tool, and an action that doesn't serve the goal is
   visible as such. _(Vanta-only.)_
3. **Local, on-device personal tuning.** `vanta tune lora` trains a local adapter from the
   operator decisions you've accepted and rejected — on-device (MPS / CUDA / CPU), nothing
   leaves the machine. Hermes also supports tuning (RL via Atropos); Vanta's distinction is
   that it's local-first and driven by _your own_ operator decisions, not a separate
   training pipeline. OpenClaw has no personal tuning.

## Where Hermes and OpenClaw lead today

We'd rather you hear it here than find out later. The honest gaps are **ecosystem and
reach**, not core capability — Vanta's channels and self-improvement loop are already
shipped (see [Comms & gateway](/comms-and-gateway) and [Self-improvement](/self-improvement)):

- **Shareable public ecosystem.** Hermes's Skills Hub and OpenClaw's plugin marketplace
  are _public_ exchanges for sharing and pulling skills/plugins. Vanta has skill
  search/install/manage, a plugin framework, and bundled skills — but no public community
  marketplace yet; its skills stay local.
- **Native mobile.** OpenClaw ships dedicated mobile nodes (iOS/Android with camera and
  voice); Vanta has no native mobile client yet (a Termux path is on the roadmap). Channel
  _count_ is now at parity — 20 adapters wired (5 verified live, rest need creds).
- **Community & maturity.** Both have a head start in public users and contributions;
  Vanta is newer.

## Migrating to Vanta

Importers between these tools are standard in this category — OpenClaw imports from Codex
CLI, Claude, and Hermes; Hermes imports from OpenClaw. Vanta is building the same:
`vanta migrate openclaw|hermes` will import your skills, MCP servers, and model config into
`~/.vanta`, with a preview → select → backup → apply flow and secrets redacted in the
report. It's a tracked next-up item on the roadmap (`VANTA-MIGRATE`).
