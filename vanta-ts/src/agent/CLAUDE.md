# CLAUDE.md — vanta-ts/src/agent

Focused map for extracted agent-loop helpers.

- `turn-loop.ts`: `runTurn` — the per-turn iteration loop (completion → tool calls → stop conditions). Emits `PostToolBatch` after resolved tool calls and `MessageDisplay` before assistant text display. Delegates the model call to `provider-call.ts`.
- `provider-call.ts`: the model-call seam extracted from turn-loop (size gate). `getCompletion` (scoped schemas → streaming-or-complete provider call) + `getCompletionWithContextRetry` (the retry policy): a **transient** provider error (idle/request timeout, 429, connection reset) is retried bounded with backoff then stopped gracefully (`{ok:false}`), a **non-transient** error fails fast (throws), a **context-window** error falls to one compaction retry. Reuses `tool-retry`'s `isTransientError`/`resolveProviderRetries` (`VANTA_PROVIDER_RETRIES` default 2, `VANTA_PROVIDER_RETRY_BACKOFF_MS` default 500). PROVIDER-HARDENING — found by the long-run stress harness.
- `auto-continue.ts`: VANTA-AUTOCONTINUE — fixes premature stopping ("agentic laziness"). `shouldAutoContinue` decides whether a would-be "done" (text, no tool calls) should keep going: fires only when the model DID work this turn, isn't awaiting the user (clarify/ask_user/trailing `?`), and either announced more work (`looksUnfinished`) or — with `VANTA_VERIFY=1` — failed the completion verifier on a done-claim. Bounded by `VANTA_AUTOCONTINUE_MAX` (default 3; `VANTA_AUTOCONTINUE=0` disables) and the turn's maxIter.
- `dispatch-tool.ts`: single tool-call execution pipeline; emits `PostToolUse` and `PostToolUseFailure`.
- `dispatch-helpers.ts`: `applySafetyGate`, retry, and output compression helpers. Emits `PermissionRequest`/`PermissionDenied`. Safety order is fixed: Rust kernel → `permissions.tsv` tightening → auto-mode classifier if enabled. PAPER-GOVERNANCE-AUDIT: every exit calls `auditGate` (best-effort, never blocks) logging a `gate` event (kernel risk + final resolution: allow/blocked/approved/denied/accept-edits-auto/delegated-auto/kernel-unreachable) via `deps.safety.logEvent`; `governance/audit.ts` reads them back for `vanta governance export`.
- `tool-scope.ts`: deferred schema subsetter; large registries expose a task-relevant subset plus `tool_search`, and recent `tool_search` result headings expand full schemas on the next provider call. `VANTA_TOOL_SCOPE=0` restores full exposure.
- `permission-gate.test.ts`: regression tests for kernel immovability, permission tightening, and auto-mode ask handling.

Do not weaken a kernel `block`; every new gate can only keep or tighten the decision.
