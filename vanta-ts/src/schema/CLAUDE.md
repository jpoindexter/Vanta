# Schema task environments

Use `TaskEnvironment` for a bounded, replayable task substrate. `runTaskStep` is side-effect-free fixture execution only; reversible or external work must enter through `commitActions`, the sole Schema API that receives a kernel executor. Use `TaskTransitionTimeline` for append-only, redacted transition events on the kernel audit chain. Use `groundTransition` and immutable revisions for interpreted task state. Install versioned model artifacts through `installTaskModel` and expose them with `inspectActiveTaskModel`; never edit a persisted version in place. Run generated transition code only through `executeTaskModel`; it fails closed without the strict platform sandbox, uses declared inputs, and records a receipt. Use `runBacktest` for complete-history certification; partial, skipped, stale, malformed, execution-failed, state-mismatched, or terminal-mismatched coverage is not certified. Reality outranks prediction: the first post-commit mismatch stops the remaining queue and opens a durable counterexample episode. Resume only after a state or model revision installs a newer immutable artifact and passes a current full-history backtest. When explanations remain ambiguous, store explicit competing hypotheses and use `planDiscriminatingProbes`; propose no more than three probes, prefer information gain with low reversible cost, and never treat missing evidence as support. Use `planCertifiedModel` only with a live certification and unchanged history; bound expanded states, distinct states, depth, and cost. A `SimulatedPlan` cannot execute itself and must be converted to requests that still pass through `commitActions`. Keep live browser, repository, operator, and cryptographic verification I/O behind injected callers.
