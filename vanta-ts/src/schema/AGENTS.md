# Schema task environments

Pure versioned task-adapter protocol: validate observations/actions/snapshots, predict and execute one transition, then verify it. `fixtures.ts` provides deterministic repo and browser adapters. `timeline.ts` records typed task outcomes through the kernel audit writer and replays them only after kernel chain verification. `grounding.ts` derives revisable, provenance-bearing state without mutating transition history. `mechanism-library.ts` promotes only currently certified rules with owned support, retained counterexamples, exact held-out replay, and the existing learning gate; failed transfer scopes a new immutable version and preserves prior evidence. Production I/O stays behind injected boundaries.
