Vanta Desktop - User Flow Report

2026-07-13 · Native desktop workflow audit and streamlining plan

Summary

Vanta has enough desktop capability for daily work, but the original arrangement made ordinary agent work look like a collection of adjacent tools. The reset keeps Vanta's kernel-gated operator model and condenses the surface into Work, Outputs, and Connect. The initial theme is not defined boot failure is repaired: source and signed-package operator-flow smokes now render the shell. The remaining release work is provider-backed turn proof, queue/steer, scoped recovery, and richer context selection.

Flow: Start and complete a task

Pattern source: flow-ai-chat + flow-app-shell · Entry point: launch or New session

Target steps

  1. 1Work opens with useful context

    The user sees the current project, selected session, model scope, and visible attached-context chips. A short empty state offers concrete task starters.

    Primary action: Describe the outcome -> new agent run

  2. 2Run state appears in the conversation

    The user message appears immediately. Vanta streams the response and shows compact, named tool steps that can expand for evidence.

    Primary action: Stop -> keeps partial work and restores control

    If an action needs permission -> show the exact action with Approve, Edit, and Reject in the thread.
  3. 3Completion points to reviewable output

    The completed run summarizes changed files, preview, links, and receipts. The active-run strip remains visible if the user navigates away.

    Primary action: Open outputs -> Outputs filtered to this run

Flow: Recover from setup or execution failure

Pattern source: flow-errors · Entry point: failed bootstrap, provider setup, or failed agent step

Target steps

  1. 1Keep work visible

    The shell, draft, and cached sessions stay available. Only the affected surface reports a failure.

    Primary action: See the fix -> scoped recovery action

  2. 2Explain the failure in task language

    Vanta states whether the provider, project files, session store, or an agent step failed and whether partial output is preserved.

    Primary action: Retry failed step -> resumes from checkpoint

    If the selected provider has no credentials -> open provider setup with the current task preserved.
  3. 3Confirm reconciliation

    When retry succeeds, Vanta reports what completed and links to the output rather than silently clearing the error.

    Primary action: Review output -> Outputs for the recovered run

Flow: Attach context and review outputs

Pattern source: flow-navigation + flow-app-shell · Entry point: composer attachment or completed run

Target steps

  1. 1Choose relevant project context

    The user opens an attachment picker from the composer and searches Recent, Changed by Vanta, Mentioned in this task, or the project tree.

    Primary action: Attach file -> removable context chip

  2. 2Review a run's output

    Outputs groups artifacts by run and lets the user filter changed files, preview, links, and receipts.

    Primary action: Open output -> contextual preview or source session

  3. 3Continue from an artifact

    An output can be attached to a new task or opened in its originating conversation without replacing the primary navigation model.

    Primary action: Continue work -> Work with artifact context attached
