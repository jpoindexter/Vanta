Vanta — Go-Live Checklist

The code is shipped (9/9 beta cards). This is the go-live session: the steps that need real tokens + a clean machine to flip the four 🔒 gates to ✅. Checkboxes save in this browser.

00

Pick the narrow beta

scope discipline
  • Decide the beta = CLI + Telegram, dev/operator tasks, one machine. Don't gate launch on matching the full channel/extension surface.
  • Write the one-sentence "done": "5 strangers install it and each gets one real task done live, without me in the loop."
01

Install on a clean machine

🔒 gate 1
  • Get a fresh machine — a $5 VPS or a clean container. Not this dev box (that's the whole point of the proof).
  • One-line install (auto-installs deps):
    curl -fsSL https://raw.githubusercontent.com/jpoindexter/Vanta/main/bootstrap.sh | bash
  • Check optional capability deps (voice/desktop/swarm):
    vanta preflight
  • Pick a model backend + key:
    vanta setup
  • Health check — must be green:
    vanta doctor
  • Prove it does a real task:
    vanta run "read README and summarize"
02

Reach you on a channel

🔒 gate 2
  • In Telegram, open @BotFather → /newbot → copy the HTTP API token.
  • Set the token (or use vanta setup messaging):
    export VANTA_TELEGRAM_TOKEN=<your-token>
  • Start the gateway (cron + channels + webhooks, one loop):
    vanta gateway
  • Message your bot → confirm it replies. That's gate 2.
  • Optional — add more channels (one gateway runs all): WhatsApp VANTA_WHATSAPP_TOKEN + VANTA_WHATSAPP_PHONE_ID; Slack VANTA_SLACK_BOT_TOKEN; Discord VANTA_DISCORD_TOKEN + VANTA_DISCORD_CHANNEL.
    Inbound for WhatsApp/Slack arrives via their webhook → point it at your gateway's webhook URL.
03

Images + voice in a channel

🔒 gate 3
  • Install whisper + ffmpeg (local speech-to-text):
    brew install openai-whisper ffmpeg
    Linux: pip install -U openai-whisper + apt install ffmpeg. Optional: export VANTA_STT_MODEL=base for better transcripts.
  • Send the bot a photo → confirm it's understood (vision).
  • Send the bot a voice memo → confirm it's transcribed. That's gate 3.
04

Runs unattended

🔒 gate 4
  • Keep the gateway alive across restarts (macOS launchd):
    vanta service install
    On a Linux VPS, run vanta gateway under systemd / a process manager instead.
  • Add a scheduled job (natural language):
    vanta cron add "8am: summarize my unread email" "0 8 * * *"
  • Confirm it fires at its time and delivers to your channel. That's gate 4.
05

Safe for strangers

already on — verify
  • Run the full proof gate — aim for all ✅:
    vanta beta-proof
    Writes docs/beta-readiness.md. The kernel already blocks rm -rf / — that's verified here.
  • Optional — restrict outbound to an allowlist (default-deny everything else):
    export VANTA_EGRESS_ALLOW=github.com,*.anthropic.com
06

Ship it

the actual launch
  • Get 5 strangers to install + each do one real task live.
  • Watch the first real session end-to-end (don't intervene — see what they hit).
  • Capture what broke → file as roadmap cards. That's the beta.
