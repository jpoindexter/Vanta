      4 · Architecture lessons to keep

      • One scheduler, not two — Hermes' swarm writes onto the kanban kernel rather than a second scheduler. Vanta's factory should fan out onto roadmap.json, not a parallel queue.
      • Aux-LLM for meta-work — Hermes uses a cheap aux model for decompose/specify/profile-describe. Mirrors Vanta's AUX-VISION; generalize via AUX-MAP.
      • Fallback as a chain, not a flag — ordered providers tried on failure beats a single primary. → MODEL-FALLBACK.
      • Curator as a daemon with pin — maintenance that consolidates + lets you freeze a good state. → SKILL-UMBRELLA.
      • Update with a rollback snapshot — never mutate a running install without a labelled way back. → SELF-UPDATE.

      5 · Where Vanta already wins

      • Enforced safety kernel (Rust assess()) — Hermes gates nothing at a boundary; this is Vanta's whole reason to exist.
      • Executive-function gates (inhibit, set-shift, closure, research-gate, self-monitor) — an ND-first operator layer Hermes has no analogue for.
      • Brain / selfhood (5D/12-axis, git-versioned) vs Hermes' flat memory provider.
      • Goal-before-tool, enforced by the kernel ledger, not a prompt rule.
      • Zero-dep kernel — the trust boundary has no supply chain of its own.

      6 · Bottom line

      Core agent parity with the customized Hermes is met, and Vanta adds a safety + executive-function spine Hermes lacks. Close the seven copy-cards (fallback, session-search, dep-audit, update, model-cmd, proxy, ACP + the DX pack + curator umbrellas + kanban-decompose) and Vanta is at full functional parity on everything that serves a single trusted operator — while deliberately declining the platform/commercial surface (Nous Portal, 21 platforms, profiles, i18n, RL datagen) that pulls against the thesis.
