Harness / System Prompt / Agent Loop

The model is not the agent.

An agent is a reasoning model inside an operating system: prompt rules, runtime harness, memory/context, tools, safety gates, and verification.

Input

User goal

The mission: what you want done.

Rules

System prompt

Identity, priorities, boundaries, style.

State

Context + memory

Files, goals, history, current screen.

Reasoning core

LLM model

Plans, predicts, decides next action. Cannot touch the world alone.

Runtime

Harness

The cockpit that feeds context, exposes tools, and records actions.

Boundary

Safety gate

Allows, asks, or blocks before risky actions run.

Hands

Tools

Files, shell, browser, email, calendar, vision, git.

Selected part

User goal

The user provides intent. The agent should translate that into a concrete goal before using tools.

Human analogy: “I need a table built.”
Agent behavior: clarify outcome, plan, act, verify.

Execution loop

01

Goal enters the agent.

02

Prompt + context shape reasoning.

03

Harness checks and runs tools.

04

World returns evidence.

05

Agent verifies and reports.

System prompt

The constitution. It says who the agent is, what matters, what to refuse, and how to behave.

Harness

The body/cockpit. It gives the model controlled access to memory, files, tools, approvals, and logs.

Agent

The whole organism. Model + prompt + harness + tools + safety + verification loop.
