Agent Architecture Map
Harness × System Prompt × Model × Tools

Who is steering?

The model is not the whole agent. An AI agent is a model placed inside a harness, constrained by a system prompt, connected to tools, and checked by safety rules.

LOOP 1

Understand active goal before choosing a tool.

LOOP 2

Model proposes next action under the system rules.

LOOP 3

Harness checks permissions and runs the tool.

LOOP 4

Tool returns evidence from the world.

LOOP 5

Agent verifies, updates state, then continues or stops.

System prompt

The system prompt is the constitution: identity, hierarchy, standards, and hard lines. It shapes behavior but cannot physically execute actions by itself.

Harness

The harness is the body and cockpit: it injects context, manages memory, exposes tools, routes approvals, and prevents unsupported actions.

Agent

The agent is the whole system in motion: model plus prompt plus harness plus tools plus verification loop.

Key idea: The model can suggest. The harness can permit. The tools can act. The system prompt tells the agent how to behave across all of it.
