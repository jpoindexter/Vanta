# Vanta v0.9.6

Vanta v0.9.6 fixes a broken trust boundary: when an operator approved creation of a new project outside the active root, the macOS shell sandbox still denied the exact command.

## Fixed

- A human-approved, direct `mkdir` now receives a temporary writable binding for its nearest existing parent directory.
- The allowance applies only to that execution. It is not persisted and does not weaken the kernel's block floor.
- Arbitrary shell syntax, `~` paths, protected paths, and directories containing credential locations remain ineligible for an extra sandbox binding.

## Verification

- Focused permission, shell, and sandbox suite: 90 tests passed.
- TypeScript typecheck: passed.
- Real macOS Seatbelt proof: an approved sibling-directory `mkdir` completed outside the project root; the empty proof directory was removed afterward.
