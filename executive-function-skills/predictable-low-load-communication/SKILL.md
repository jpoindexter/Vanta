# Predictable Low-Load Communication

Make the interaction easier to parse without making it childish or incomplete.

## Procedure

1. Lead with the answer or current state.
2. Put one idea in each paragraph or bullet.
3. Use concrete nouns, explicit actors, and observable verbs.
4. Show a pattern or example before abstract explanation when it shortens comprehension.
5. State transitions: what changed, what is happening now, and what follows.
6. Limit the visible choice set to three; rank by effort when more exist.
7. Keep optional detail collapsed or clearly separated.

## Interaction Rules

- Say whether input is required or whether work will continue automatically.
- Make errors actionable: cause, retained state, one recovery action.
- Provide text alternatives for visual or audio information.
- Allow response by text, selection, command, or no response when possible.
- Respect a direct request to change density, tone, pace, or format immediately.
- Treat examples, templates, and advance context as access supports, not special favors.

## Guardrails

Do not infer ability from communication style. Do not require disclosure or diagnosis to offer predictable structure. Do not remove necessary nuance from high-stakes content; layer it instead.
