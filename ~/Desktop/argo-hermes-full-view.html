Hermes has broader surface area and older UX/platform breadth; Argo has the safer core and most high-value parity pieces.

What is missing / weak

  • Session search: Hermes has FTS-style session discovery; Argo lists/resumes but does not deeply search old conversations.
  • @-file mentions: Claude/Hermes-style path autocomplete is still a high-value UX gap.
  • Messaging breadth: Argo has Telegram + webhooks, not the long tail of Slack/Discord/Signal/Matrix/etc.
  • Desktop/web UI: Argo is TUI-first; a dashboard/cockpit for normal daily operation is parked.
  • Skill bundles: individual skills work; YAML bundles for composite operator modes are pending.
  • Heartbeat: brain/selfhood updates are not yet wired to continuous daemon ticks.
  • Prefer-local routing: directive exists, but automatic simple-task routing to Ollama is not finished.
  • Compartmentalized self-repair: factory has protected paths and autonomy ladder; formal body-tier caps are pending.
  • Secret read hygiene: readable sibling zones need a stronger secret-filename guard.

Red lines

Do not copy Hermes where Hermes is unsafe. Replace advisory file safety with Argo’s kernel boundary every time. Never weaken scope enforcement for convenience.
  • No destructive commands.
  • No secrets in commits.
  • No outbound comms without approval.
  • No autonomous edits to kernel/factory/manifesto.
  • No claims without verified output.

Bottom line

Argo does not need to become a larger Hermes clone. It should remain the safer operator: kernel-enforced, goal-aware, verified, neurodivergent-first, and visually/technically coherent. The next improvements should target daily friction and trust bottlenecks, not raw feature count.

Core Hermes parity: high. Surface breadth and polish: still open.
